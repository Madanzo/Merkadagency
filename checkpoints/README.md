# Checkpoints

Development milestones documenting project state at key points.

## Naming Convention

```
YYYY-MM-DD_description.md
```

Examples:
- `2026-01-16_homepage-complete.md`
- `2026-01-20_crm-integration.md`
- `2026-02-01_v1-launch.md`

## Template

Each checkpoint should include:

```markdown
# Checkpoint: [Description]

**Date:** YYYY-MM-DD
**Version:** vX.Y.Z (if applicable)
**Commit:** [commit hash]

## What's Working
- Feature 1
- Feature 2

## What's Pending
- [ ] Upcoming task 1
- [ ] Upcoming task 2

## Blockers
- Issue description (if any)

## Notes
Additional context or decisions made.
```
